# SPOT 방송부장 운영 가이드

이 문서는 **새로운 방송부장(LEADER)** 이 SPOT를 바로 운영할 수 있도록, 화면 기준 업무 흐름과 시스템 규칙을 정리한 안내서입니다.

## 1. 서비스 한눈에 보기

SPOT은 방송부 노래 신청을 한곳에서 관리하는 시스템입니다.

- 학생(USER): 기상송/점심방송 신청, 본인 `PENDING` 신청 취소
- 방송부원(MEMBER): 점심방송 승인/거절/일괄 승인, YouTube 재생목록 내보내기, 기상송 대기열 삭제
- 방송부장(LEADER): 위 기능 + 기상송 스케줄 승인/변경/승인 취소 + 사용자/YouTube/신청 안내/감사 로그 관리

## 1-1. 인수인계 직후 먼저 확인할 것

- 내 계정 역할이 `LEADER`인지 확인
- YouTube 탭에서 방송부 계정 연결 상태 확인
- 비상 대응용으로 다른 운영진 1명 이상을 `MEMBER` 이상으로 지정

## 2. 방송부장이 알아야 할 핵심 운영 규칙 (KST)

### 기상송

- 매일 `08:00`에 자동 승인 로직 실행
- 자동 승인 대상 날짜: **3일 뒤**
- 해당 날짜 슬롯은 최대 2곡
- 승인 우선순위: `PENDING` 생성 순(`created_at` 오름차순)
- 승인된 곡은 재생일이 지나면 자동 삭제됨
- `PENDING`(대기중)은 자동 삭제되지 않음

### 점심방송

- 신청은 `PENDING`으로 쌓임
- 방송부 승인 시 `APPROVED`로 변경되고 `play_date`는 승인 시각으로 기록
- 거절된 곡은 7일 후 자동 삭제 (신청자의 마이페이지에서)
- `PENDING`(대기중)은 자동 삭제되지 않음

### 자동 정리 스케줄

- 매시 `10분`(예: 00:10, 01:10, ...)에 정리 실행
- 삭제 대상:
  - `APPROVED` + `play_date < 오늘 00:00`
  - `REJECTED` + 거절 시점 기준 7일 경과
- 서버 시작 시에도 1회 즉시 정리 실행

## 3. 관리자 화면 탭별 업무

### 3-1. 기상송 탭 (LEADER 전용)

- `기상송 대기열` 확인
- 곡별 처리:
  - `승인`: 날짜 지정 승인(스케줄 확정)
  - `삭제`: 대기열에서 즉시 제거
- `승인된 기상송 스케줄`에서:
  - 날짜 재조정 가능
  - 승인 취소 시 다시 `PENDING`으로 복귀

주의:
- 과거 날짜로 스케줄 지정 불가
- 기준 날짜는 단순 자정이 아니라 기상송 표시 규칙(08:00 경계)을 따름

### 3-2. 점심방송 탭 (MEMBER/LEADER)

- `점심방송 신청 목록`: `PENDING` 목록
- 처리 방법:
  - 개별 `승인`/`거절`
  - `일괄 승인`(대기중 전체)
- `오늘 방송 목록`에서 승인 곡 확인
- YouTube 연동 상태가 연결됨일 때 `YouTube 재생목록 내보내기` 사용 가능

### 3-3. 사용자 관리 탭 (LEADER 전용)

- 이름/이메일로 사용자 검색
- 사용자 역할 변경: `USER`, `MEMBER`, `LEADER`
- 사용자 차단/해제는 일반 사용자(USER)만 가능
- `MEMBER`, `LEADER`는 차단 불가
- 본인 LEADER 권한을 직접 내리는 변경은 차단됨

### 3-4. YouTube 탭 (LEADER 전용)

- 방송부 YouTube 계정 연결 상태 확인
- OAuth 연결 시작/재연결
- 연결이 끊겨 있으면 재생목록 내보내기 기능이 동작하지 않음

### 3-5. 신청 안내 탭 (LEADER 전용)

- 학생 신청 페이지 안내 문구 3종 관리:
  - 기상송 안내
  - 점심방송 안내
  - 공통 안내
- 각 문구 제약:
  - 빈 문자열 불가
  - 200자 이하

### 3-6. 감사 탭 (LEADER 전용)

- 관리자 API 변경 요청 중심 로그 확인
- 검색 기준: 이름/이메일/경로/메서드
- 확인 가능 항목:
  - 요청 시각, 사용자, 역할, 메서드/경로, 상태코드, 처리 시간, 요청 query/body
- `304` 응답은 감사 로그 대상에서 제외

## 4. 일일 운영 체크리스트

### 4-1. 아침 (08:00 전후)

- 기상송 자동 승인 결과 확인
- 스케줄 충돌/누락 시 수동으로 날짜 재조정

### 4-2. 점심방송 전

- 점심방송 신청 목록 확인
- 부적절한 신청 거절, 나머지 승인(필요 시 일괄 승인)
- 오늘 방송용 YouTube 재생목록 내보내기

### 4-3. 수시

- 악성 신청 발생 시 사용자 차단
- 신청 안내 문구 최신화
- 감사 로그 확인(이상 요청/오작동 추적)

## 5. 자주 헷갈리는 포인트

- `PENDING`은 날짜가 지나도 자동 삭제되지 않습니다.
- 자정(00:00) 정각 삭제 작업은 없습니다. 정리는 매시 10분에 수행됩니다.
- 기상송 화면 기준 날짜는 08:00을 경계로 오늘/내일이 바뀝니다.
- 점심방송 “오늘 방송 목록”은 `play_date`가 오늘인 `APPROVED`만 표시됩니다.

## 6. 운영 장애 시 우선 확인 순서

1. 사용자 권한이 맞는지 확인 (`LEADER`/`MEMBER`/`USER`)
2. 신청 상태가 맞는지 확인 (`PENDING`만 승인/거절 가능 등)
3. YouTube 연결 상태 확인 (재생목록 내보내기 실패 시)
4. 감사 로그에서 실패 요청의 응답 코드와 요청 body 확인
